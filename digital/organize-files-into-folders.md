## Goal

Your files are grouped into a simple folder structure with consistent names, useful dates, and no giant "misc" pile that becomes impossible to search.

## Preconditions

- Access to the computer, cloud drive, or phone storage you want to organize.
- Enough free space to move files without syncing errors.
- A backup if you are reorganizing important documents.

## Steps

1. **Pick the root location.** Choose one main place, such as Documents, a cloud drive, or an external drive, for the files you are organizing. → *Expect:* you know where the top-level folders will live.
2. **Create a top-level scheme.** Make broad folders such as Admin, Work, School, Home, Finance, Photos, and Archive. → *Expect:* every common file type has an obvious first destination.
3. **Use consistent names.** Rename folders with plain words and avoid duplicates like Work, work stuff, and Job files. → *Expect:* similar files point to the same folder.
4. **Add dates where they help.** Use `YYYY-MM-DD` or `YYYY-MM` at the start of records, exports, scans, and project snapshots. → *Expect:* dated files sort correctly by name.
5. **Limit nesting.** [BRANCH: broad folder | project folder] use two or three levels for ongoing work, and create deeper folders only for large projects. → *Expect:* you can reach common files in a few clicks.
6. **Sort the loose files.** Move desktop, downloads, and old cloud-drive clutter into the chosen folders. → *Expect:* loose-file areas shrink substantially.
7. **Replace misc with a decision.** If tempted to make Misc, choose Archive, To Sort, or a specific category with a review date. → *Expect:* there is no permanent catch-all folder.
8. **Search-test the system.** Look for one tax file, one photo, and one active document using folder browsing and search. → *Expect:* each file is findable without guessing several places.

## Decision points

- You need files across devices → use a cloud drive as the root and keep large archives local only if storage is limited.
- You share folders with others → prefer names that make sense to the group, not private shorthand.
- A folder has more than about 100 mixed files → split by project, year, client, or document type.

## Failure modes & recovery

- **F1 Sync conflict appears:** detect duplicate files with conflict names, recover by comparing modified dates and keeping the newest correct version.
- **F2 File links break:** detect when app shortcuts or recent files fail, recover by reopening the moved file from its new folder and updating shortcuts.
- **F3 Too many categories:** detect when you hesitate before filing each item, recover by merging categories into broader folders.
- **F4 To Sort grows forever:** detect when it becomes another Misc folder, recover by scheduling a weekly review and moving or deleting each item.

## Verification

A new file can be named, dated if useful, and placed in one clear folder without creating or using a permanent Misc folder.

## Variations

- `photos`: organize by year and event, and let the photo app keep albums separate from the original library.
- `small-business`: top folders often work best by year, client, finance, legal, and operations.
- `shared-drive`: document naming rules in a short README so collaborators do not invent new categories.

## Safety & privacy

Moving files can affect sync, shortcuts, shared links, and backups. Do not delete originals until you confirm the destination is synced and backed up, especially for identity, tax, legal, and medical records.
